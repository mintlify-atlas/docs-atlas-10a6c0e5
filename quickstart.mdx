---
title: "Quickstart"
description: "Get Social Media Fetcher running and download your first media gallery in under 5 minutes."
---

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/Omarleel/Social-Media-Fetcher.git
    cd Social-Media-Fetcher
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    npm install
    ```

    <Note>
      The `postinstall` script runs `puppeteer browsers install chrome` automatically after `npm install` completes. This downloads the Chromium browser that Puppeteer uses for browser-automation engines (X, TikTok, Instagram, Threads, DeviantArt, OnlyFans). It may take a minute on first install.
    </Note>
  </Step>

  <Step title="Configure your environment">
    Copy the example file and fill in the required values:

    ```bash
    cp .env.example .env
    ```

    At minimum, set the three server variables and at least one platform token. For example, to use the X (Twitter) engine:

    ```env .env
    PORT=3000
    DIR_STORAGE=./storage
    THREADS_DOWNLOAD=5

    X_AUTH_TOKEN=your_auth_token_cookie_value
    ```

    <Tip>
      On low-resource machines (limited RAM or CPU), set `THREADS_DOWNLOAD=3` to keep concurrent downloads manageable and avoid network saturation or thermal throttling.
    </Tip>

    See [Configuration](/configuration) for a full list of all environment variables and instructions on finding your authentication cookies.
  </Step>

  <Step title="Start the server">
    ```bash
    npm start
    ```

    Or directly:

    ```bash
    node app.js
    ```

    The server listens on the port defined in `PORT` (default `3000`). You should see:

    ```
    🚀 Servidor SocialMediaFetcher corriendo en puerto 3000
    ```
  </Step>

  <Step title="Make your first request">
    Download the 10 most recent media items from an X (Twitter) profile:

    ```bash
    curl "http://localhost:3000/x/get-all-media?username=elonmusk&limit=10"
    ```

    A successful response looks like this:

    ```json
    {
      "status": "success",
      "profile": {
        "username": "elonmusk",
        "name": "Elon Musk",
        "avatar": "https://pbs.twimg.com/profile_images/.../photo.jpg"
      },
      "total_requested": 10,
      "total_processed": 10,
      "downloads": [
        {
          "url": "https://pbs.twimg.com/media/...",
          "filename": "photo.jpg",
          "path": "./storage/x/elonmusk/images/photo.jpg"
        }
      ]
    }
    ```

    Downloaded files are saved to `DIR_STORAGE/x/elonmusk/` on disk.
  </Step>
</Steps>
