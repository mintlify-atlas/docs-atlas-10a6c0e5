---
title: "X (Twitter)"
description: "Download all media from an X (Twitter) user's media tab."
---

## Endpoint

```
GET /x/get-all-media
```

## Authentication

Set the following variable in your `.env` file:

| Variable | Description |
|---|---|
| `X_AUTH_TOKEN` | Value of the `auth_token` cookie from a logged-in X session |

<Note>
  The token is injected as a browser cookie before navigation begins. X requires an authenticated session to access the full `UserTweets` and `UserMedia` GraphQL responses.
</Note>

## Query parameters

<ParamField query="username" type="string" required>
  The X username of the account (with or without `@`). The `@` prefix is stripped automatically.
</ParamField>

<ParamField query="limit" type="integer">
  Maximum number of distinct media items to collect. Collection stops once this count is reached.
</ParamField>

<ParamField query="method" type="string" default="normal">
  Download strategy. Accepted values:

  - `normal` ŌĆö downloads each media item in real-time as it is intercepted during scrolling.
  - `batch` ŌĆö collects all media URLs first by scrolling to completion, then downloads them concurrently using a parallel worker pool.
</ParamField>

## curl example

```bash
curl "http://localhost:3000/x/get-all-media?username=exampleuser&limit=100&method=batch"
```

## Notes

<Note>
  SMF intercepts `UserTweets` and `UserMedia` GraphQL network responses during automated scrolling to extract media URLs and profile metadata. No separate API calls are made.
</Note>

<Note>
  Stagnation detection stops the scroll loop after **3 consecutive scroll iterations** that produce no new media items. This handles accounts where the end of the media tab has been reached or the session has been rate-limited.
</Note>

<Tip>
  Use `method=batch` when downloading large archives. It collects all URLs before starting downloads, which allows the full concurrency pool to be used from the start. Use `method=normal` when you want downloads to begin immediately during collection.
</Tip>

<Warning>
  Media files are separated into `images/` and `videos/` subfolders within the user's storage directory. Video files use the `.mp4` extension; all other media uses the detected extension.
</Warning>

## Error codes

| HTTP status | `error` value | Cause |
|---|---|---|
| 500 | Any message | Browser launch failure, authentication error, or network problem. Check that `X_AUTH_TOKEN` is set and valid. |
